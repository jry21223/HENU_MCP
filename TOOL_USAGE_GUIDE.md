# 如何让AI模型优先调用工具而不是编造答案

## 问题描述

有些AI模型倾向于"幻觉"（编造信息）而不是调用MCP工具获取真实数据。这会导致：
- 返回错误的课表信息
- 编造不存在的预约记录
- 假装执行了操作但实际没有

## 解决方案

### 1. 工具描述优化（已完成）

我们已经为所有工具添加了明确的描述，包含：
- **【必须调用】** 标记 - 明确告诉模型这是必须调用的工具
- **功能说明** - 清楚说明工具的作用
- **重要提示** - 强调不要编造信息

示例：
```python
def current_course(...):
    """
    【必须调用】查询当前课程状态 - 获取正在上的课和下一节课
    
    功能：基于当前时间和课表，返回：
    - 当前正在上的课程（如果有）
    - 下一节课的信息
    
    重要：不要猜测或编造课程信息，必须调用此工具获取准确的实时数据。
    """
```

### 2. 在AI客户端中添加系统提示

在Cherry Studio或其他MCP客户端中，添加以下系统提示：

```
你是河南大学校园助手。当用户询问课表、图书馆预约等信息时：

【重要规则】
1. 必须调用相应的MCP工具获取真实数据
2. 不要编造或猜测任何信息
3. 如果工具调用失败，明确告诉用户失败原因
4. 不要假装执行了操作，必须等待工具返回结果

【工具使用指南】
- 查询课表 → 必须调用 sync_schedule 或 latest_schedule
- 查询当前课程 → 必须调用 current_course
- 查看图书馆区域 → 必须调用 library_locations
- 图书馆预约 → 必须调用 library_reserve
- 查询预约记录 → 必须调用 library_records
- 取消预约 → 必须调用 library_cancel

【禁止行为】
- ❌ 不要说"根据你的课表..."然后编造内容
- ❌ 不要说"预约成功"除非工具返回成功
- ❌ 不要编造课程时间、地点、教师等信息
- ❌ 不要编造图书馆区域名称或座位号
- ❌ 不要假装已经执行预约操作
- ✅ 总是说"让我查询一下..."然后调用工具
- ✅ 预约前先调用 library_locations 查看可用区域
```

### 3. 使用特定的提示词模板

当与AI对话时，使用这些提示词可以提高工具调用率：

**查询课表：**
```
请使用MCP工具查询我今天的课表
```

**查询当前课程：**
```
调用工具查看我现在在上什么课
```

**图书馆预约：**
```
使用工具帮我预约图书馆座位
先查询可用区域，然后预约一楼东的001座位
```

**查询预约记录：**
```
调用工具查看我的图书馆预约记录
```

### 4. 模型选择建议

不同模型对工具调用的支持程度不同：

**推荐模型（工具调用能力强）：**
- Claude 3.5 Sonnet
- GPT-4 Turbo
- GPT-4o
- Gemini 1.5 Pro

**需要额外提示的模型：**
- Claude 3 Opus（需要明确提示）
- GPT-3.5 Turbo（可能需要多次提示）
- 其他开源模型（建议在系统提示中强调）

### 5. Cherry Studio 配置建议

在Cherry Studio中：

1. **设置系统提示**
   - 进入设置 → 助手配置
   - 添加上面的系统提示

2. **启用工具调用**
   - 确保MCP服务器已连接
   - 在对话中可以看到可用工具列表

3. **使用明确的指令**
   - 开始对话时说："请使用MCP工具查询..."
   - 如果模型编造信息，提醒它："请调用工具而不是猜测"

### 6. 验证工具是否被调用

正确的对话流程应该是：

```
用户：我今天有什么课？

AI：让我查询一下你的课表...
[调用 current_course 工具]
[工具返回结果]

AI：根据查询结果，你今天有以下课程：
- 第3-4节：高等数学，张老师，教学楼A101
- 第6-7节：英语，李老师，教学楼B203
```

**错误的流程（编造信息）：**
```
用户：我今天有什么课？

AI：根据你的课表，你今天有高等数学和英语课...
[没有调用任何工具，直接编造]
```

### 7. 故障排除

如果模型仍然不调用工具：

1. **检查MCP连接**
   ```bash
   # 运行诊断工具
   python3 diagnose_mcp.py
   ```

2. **查看工具列表**
   - 在Cherry Studio中应该能看到所有可用工具
   - 如果看不到，说明MCP服务器未正确连接

3. **使用更明确的指令**
   - 不要说："我今天有什么课？"
   - 改说："请调用current_course工具查询我今天的课程"

4. **切换模型**
   - 如果当前模型不支持工具调用
   - 尝试切换到推荐的模型

## 总结

通过以上方法的组合使用，可以大大提高AI模型调用工具的概率：

1. ✅ 优化的工具描述（已完成）
2. ✅ 添加系统提示（需要在客户端配置）
3. ✅ 使用明确的指令（用户习惯）
4. ✅ 选择合适的模型（模型能力）

如果问题仍然存在，可以在GitHub提issue或联系开发者。


## 图书馆预约工具特别说明

### 预约流程

正确的预约流程应该包含以下步骤：

1. **查询可用区域**
   ```
   用户：我想预约图书馆座位
   AI：让我先查询可用的图书馆区域...
   [调用 library_locations 工具]
   AI：目前可预约的区域有：一楼东、一楼西、二楼东...
   ```

2. **执行预约**
   ```
   用户：预约一楼东的001座位
   AI：好的，我来帮你预约...
   [调用 library_reserve 工具]
   AI：预约成功！座位：一楼东-001，日期：2026-03-12
   ```

3. **查询预约记录**
   ```
   用户：查看我的预约记录
   AI：让我查询一下...
   [调用 library_records 工具]
   AI：你有以下预约记录：...
   ```

### 常见错误示例

❌ **错误1：编造区域名称**
```
用户：有哪些区域可以预约？
AI：图书馆有一楼、二楼、三楼可以预约...
[没有调用工具，编造信息]
```

✅ **正确做法：**
```
用户：有哪些区域可以预约？
AI：让我查询一下可用区域...
[调用 library_locations 工具]
AI：根据查询结果，可预约区域有：...
```

❌ **错误2：假装预约成功**
```
用户：帮我预约一楼东001座位
AI：好的，已经帮你预约成功了！
[没有调用工具，假装成功]
```

✅ **正确做法：**
```
用户：帮我预约一楼东001座位
AI：好的，我来执行预约...
[调用 library_reserve 工具]
AI：预约成功/失败：[工具返回的实际结果]
```

❌ **错误3：编造预约记录**
```
用户：我有哪些预约？
AI：你预约了明天上午的座位...
[没有调用工具，编造记录]
```

✅ **正确做法：**
```
用户：我有哪些预约？
AI：让我查询你的预约记录...
[调用 library_records 工具]
AI：查询到以下预约：[工具返回的实际记录]
```

### 系统提示增强版（针对预约功能）

在Cherry Studio中添加以下系统提示：

```
关于图书馆预约的特别规则：

1. 预约前必须先调用 library_locations 查询可用区域
2. 执行预约时必须调用 library_reserve，不能假装
3. 只有工具返回 success: true 才能说预约成功
4. 查询记录必须调用 library_records，不能编造
5. 取消预约必须调用 library_cancel，不能假装

禁止说：
- "已经帮你预约好了"（没调用工具）
- "预约成功"（工具返回失败）
- "你有以下预约"（没调用工具）

必须说：
- "让我查询可用区域..."（然后调用工具）
- "我来执行预约..."（然后调用工具）
- "让我查询预约记录..."（然后调用工具）
```

### 验证清单

在使用预约功能时，确保：

- [ ] 查询区域时调用了 library_locations
- [ ] 预约时调用了 library_reserve
- [ ] 查询记录时调用了 library_records
- [ ] 取消时调用了 library_cancel
- [ ] AI没有编造任何信息
- [ ] AI等待工具返回结果后才回复
- [ ] 预约结果与工具返回一致

如果发现AI没有调用工具，立即提醒：
```
请不要编造信息，必须调用MCP工具获取真实数据
```
