# 如何让AI模型优先调用工具而不是编造答案

## 问题描述

有些AI模型倾向于"幻觉"（编造信息）而不是调用MCP工具获取真实数据。这会导致：
- 返回错误的课表信息
- 编造不存在的预约记录
- 假装执行了操作但实际没有

## 解决方案

### 1. 工具描述优化（已完成）

我们已经为所有工具添加了明确的描述，包含：
- **【必须调用】** 标记 - 明确告诉模型这是必须调用的工具
- **功能说明** - 清楚说明工具的作用
- **重要提示** - 强调不要编造信息

示例：
```python
def current_course(...):
    """
    【必须调用】查询当前课程状态 - 获取正在上的课和下一节课
    
    功能：基于当前时间和课表，返回：
    - 当前正在上的课程（如果有）
    - 下一节课的信息
    
    重要：不要猜测或编造课程信息，必须调用此工具获取准确的实时数据。
    """
```

### 2. 在AI客户端中添加系统提示

在Cherry Studio或其他MCP客户端中，添加以下系统提示：

```
你是河南大学校园助手。当用户询问课表、图书馆预约等信息时：

【重要规则】
1. 必须调用相应的MCP工具获取真实数据
2. 不要编造或猜测任何信息
3. 如果工具调用失败，明确告诉用户失败原因
4. 不要假装执行了操作，必须等待工具返回结果

【工具使用指南】
- 查询课表 → 必须调用 sync_schedule 或 latest_schedule
- 查询当前课程 → 必须调用 current_course
- 图书馆预约 → 必须调用 library_reserve
- 查询预约记录 → 必须调用 library_records
- 取消预约 → 必须调用 library_cancel

【禁止行为】
- ❌ 不要说"根据你的课表..."然后编造内容
- ❌ 不要说"预约成功"除非工具返回成功
- ❌ 不要编造课程时间、地点、教师等信息
- ✅ 总是说"让我查询一下..."然后调用工具
```

### 3. 使用特定的提示词模板

当与AI对话时，使用这些提示词可以提高工具调用率：

**查询课表：**
```
请使用MCP工具查询我今天的课表
```

**查询当前课程：**
```
调用工具查看我现在在上什么课
```

**图书馆预约：**
```
使用工具帮我预约图书馆座位
```

### 4. 模型选择建议

不同模型对工具调用的支持程度不同：

**推荐模型（工具调用能力强）：**
- Claude 3.5 Sonnet
- GPT-4 Turbo
- GPT-4o
- Gemini 1.5 Pro

**需要额外提示的模型：**
- Claude 3 Opus（需要明确提示）
- GPT-3.5 Turbo（可能需要多次提示）
- 其他开源模型（建议在系统提示中强调）

### 5. Cherry Studio 配置建议

在Cherry Studio中：

1. **设置系统提示**
   - 进入设置 → 助手配置
   - 添加上面的系统提示

2. **启用工具调用**
   - 确保MCP服务器已连接
   - 在对话中可以看到可用工具列表

3. **使用明确的指令**
   - 开始对话时说："请使用MCP工具查询..."
   - 如果模型编造信息，提醒它："请调用工具而不是猜测"

### 6. 验证工具是否被调用

正确的对话流程应该是：

```
用户：我今天有什么课？

AI：让我查询一下你的课表...
[调用 current_course 工具]
[工具返回结果]

AI：根据查询结果，你今天有以下课程：
- 第3-4节：高等数学，张老师，教学楼A101
- 第6-7节：英语，李老师，教学楼B203
```

**错误的流程（编造信息）：**
```
用户：我今天有什么课？

AI：根据你的课表，你今天有高等数学和英语课...
[没有调用任何工具，直接编造]
```

### 7. 故障排除

如果模型仍然不调用工具：

1. **检查MCP连接**
   ```bash
   # 运行诊断工具
   python3 diagnose_mcp.py
   ```

2. **查看工具列表**
   - 在Cherry Studio中应该能看到所有可用工具
   - 如果看不到，说明MCP服务器未正确连接

3. **使用更明确的指令**
   - 不要说："我今天有什么课？"
   - 改说："请调用current_course工具查询我今天的课程"

4. **切换模型**
   - 如果当前模型不支持工具调用
   - 尝试切换到推荐的模型

## 总结

通过以上方法的组合使用，可以大大提高AI模型调用工具的概率：

1. ✅ 优化的工具描述（已完成）
2. ✅ 添加系统提示（需要在客户端配置）
3. ✅ 使用明确的指令（用户习惯）
4. ✅ 选择合适的模型（模型能力）
